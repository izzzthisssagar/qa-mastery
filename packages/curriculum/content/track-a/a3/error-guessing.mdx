## Bugs hide where the spec stays silent

Every formal technique tests what the spec describes. Error guessing tests what the spec forgot. A good tester has a mental list of inputs that break software again and again, and reaches for it the moment the systematic tests pass. It is the cheapest technique you own and often the highest-yield.

## See it — the tester's instinct list

These are inputs experienced testers try without being told to. Most apps fail at least one.

| Trigger | Why it breaks things |
|---|---|
| Empty / whitespace-only input | Validation often checks `null` but not `"   "` |
| Leading / trailing spaces | `" admin@x.com "` saved or rejected inconsistently |
| Special chars `' " < > & %` | SQL, HTML, and encoding bugs |
| Paste vs type | Paste skips key-by-key validation |
| Double-click submit | Duplicate orders, double charges |
| Browser Back after submit | Stale form, resubmitted POST |
| Huge input (10k chars) | Layout breaks, DB column overflows |

## Try it — match the problem to the technique

You decide which technique fits before you write a single case. Walk these:

- Age must be 18-65. One field, a numeric range, edges matter -> Boundary Value Analysis.
- Email is valid or invalid. Distinct classes treated the same -> Equivalence Partitioning.
- Free shipping applies if (member AND cart > $50) OR coupon. Several conditions combine -> Decision Table.
- An order moves Pending -> Paid -> Shipped -> Delivered. Behaviour depends on current state -> State Transition.
- "I bet the discount field chokes on a negative number." No model, just a hunch -> Error Guessing.

The skill is not knowing each technique. It is reading the problem shape and picking fast.

## Do it — an error-guessing checklist for BuggyShop

Open BuggyShop and brainstorm a checklist for signup plus checkout. Do not test against the spec; test against your instincts. Aim for 12+ items, for example:

- Sign up with an email that has trailing spaces, then log in without them.
- Paste a 500-character name into the name field.
- Submit signup twice fast (double-click) and check for duplicate accounts.
- Enter `'; DROP TABLE` style and emoji into address fields.
- Add an item, hit Back after payment, see if the order resubmits.
- Set quantity to 0, then to a negative number, then to 99999.
- Apply the same coupon twice.

For each, note what you expected versus what happened. Where a checklist item maps cleanly onto a range, a set of classes, or combined conditions, swap in the formal technique and capture the gaps as bugs. A richer interactive version of this checklist is planned.
