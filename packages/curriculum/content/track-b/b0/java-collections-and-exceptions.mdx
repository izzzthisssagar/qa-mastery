## Holding many things, and handling things going wrong

Two more building blocks and you're ready for Selenium. First, **collections** —
because a test rarely deals with one element. Second, **exceptions** — because
automation lives in a world where elements vanish, pages stall, and the
unexpected is routine.

## See it — List and Map

A **List** (commonly `ArrayList`) is an ordered, resizable sequence accessed by
index. `findElements` hands you exactly this:

```java
List<WebElement> rows = driver.findElements(By.tagName("tr"));
for (WebElement row : rows) {           // for-each loop
    System.out.println(row.getText());
}
System.out.println("Found " + rows.size() + " rows");
```

A **Map** (commonly `HashMap`) stores **key-value** pairs — perfect for test data
you look up by name:

```java
Map<String, String> creds = new HashMap<>();
creds.put("username", "sagar");
creds.put("password", "secret");
String user = creds.get("username");    // "sagar"
```

Keys are unique and unordered; you fetch a value by its key with `get`.

## Try it — iterate

The for-each loop (`for (WebElement row : rows)`) reads each item in turn — no
index bookkeeping. For a Map you can loop its `keySet()` or `entrySet()`. The
point: collections plus a loop is how you assert on *many* elements at once
("every price is positive", "the table has 5 rows").

## Exceptions — when the unexpected is normal

If Selenium can't find an element, it **throws** a `NoSuchElementException`. Left
unhandled, your program crashes and the browser stays open. `try/catch/finally`
keeps you in control:

```java
try {
    driver.findElement(By.id("checkout")).click();   // risky
} catch (NoSuchElementException e) {
    System.out.println("Checkout button missing — test fails cleanly");
} finally {
    driver.quit();                                    // always runs
}
```

`try` wraps the risky call; `catch` handles the specific failure; `finally`
**always** runs — the natural home for cleanup like `driver.quit()`. This is why
exceptions matter to a tester: they turn a crash into a *reported, cleaned-up
failure*.

## Do it — written exercise

Write a `HashMap<String,String>` of two test users, then a for-each loop that
prints each username. Separately, wrap a `findElement(...).click()` in
try/catch/finally where the `finally` closes the browser. In one sentence, explain
why putting `driver.quit()` in `finally` is safer than putting it after the
`try`.
