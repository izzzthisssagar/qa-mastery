## The build tool that fetches your dependencies

You need Selenium and TestNG on the classpath to compile a test. Downloading
jar files by hand and wiring them up is how projects rot. Maven does it for
you: you declare what you need in `pom.xml`, and Maven fetches the exact
versions and makes them available.

## See it — dependencies in pom.xml

```xml
<dependencies>
  <dependency>
    <groupId>org.seleniumhq.selenium</groupId>
    <artifactId>selenium-java</artifactId>
    <version>4.20.0</version>
  </dependency>
  <dependency>
    <groupId>org.testng</groupId>
    <artifactId>testng</artifactId>
    <version>7.10.2</version>
    <scope>test</scope>
  </dependency>
</dependencies>
```

Maven also fixes the layout: source in `src/main/java`, tests in
`src/test/java`. Keep to it and the IDE and command line both just work.

## Try it — a suite with testng.xml

A `testng.xml` file picks **which** tests run. You can list classes, whole
packages, or filter by group:

```xml
<suite name="Regression">
  <test name="login">
    <groups><run><include name="smoke"/></run></groups>
    <classes>
      <class name="tests.LoginTest"/>
    </classes>
  </test>
</suite>
```

Tag tests with `@Test(groups = "smoke")` and the suite above runs only the
smoke tests — a fast subset — while a full run includes everything. Run it all
from the command line with one command:

```
mvn test
```

No IDE needed, which is exactly what a CI server runs on every push.
