## A bug report is a request someone has to act on

You found the bug. Good. But finding it is half the job. If the developer reads your report and shrugs, the bug stays alive. A report is not a diary entry, it is a work order. The best testers are judged less on how many bugs they find and more on how many get fixed because the report was clear enough to act on.

## See it — the same bug, two reports

One bug: on BuggyShop, applying coupon `SAVE10` to an empty cart crashes the page.

| Vague report | Report that gets fixed |
|---|---|
| Title: "Coupon broken" | Title: "Applying SAVE10 to an empty cart throws 500 error" |
| "It doesn't work sometimes." | Env: Chrome 125, Win 11, staging build 4.2.1 |
| No steps | Steps: 1) Open cart with 0 items 2) Enter `SAVE10` 3) Click Apply |
| No expected/actual | Expected: "Cart is empty" message. Actual: 500 error page |
| No evidence | Screenshot + server log line attached |

The left one bounces back with "works for me." The right one gets assigned in minutes. Same bug, opposite outcome.

## Try it — fix this report in your head

A teammate files: "Checkout is slow and the total looks wrong, also the email never came." Read it the way a developer would.

There are three separate problems buried here: slowness, a wrong total, and a missing email. None has steps, an environment, or numbers. A developer can't tell which to fix first, can't reproduce any of them, and a single fix can't close all three. Split it into three reports. For the wrong total, you would write: steps to reach checkout, the exact items and prices, expected total (say $42.00), actual total ($420.00), and a screenshot. Now it is reproducible and actionable.

## Do it — file a real one on BuggyShop

Open BuggyShop and trigger a genuine defect (the equivalence-partitioning and boundary-value labs each have a known input that misbehaves). Write the report yourself: a title that names the action and the symptom, your browser and build, numbered steps a stranger could follow, expected vs actual on separate lines, and one piece of evidence. Then hand it to a friend and ask them to reproduce it using only your words. If they can't, your report isn't done. A graded interactive version of this lab is planned in the EP and BVA lessons.
