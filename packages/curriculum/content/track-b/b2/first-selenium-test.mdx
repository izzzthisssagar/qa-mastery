## A test is just four moves

Strip a Selenium test to its bones and it is always the same shape: open a
browser, go to a page, read or do something, then shut the browser. Everything
else in this module — finding elements, clicking, waiting — fits inside that
frame. Learn the frame first.

## See it — the shape of a test

```java
WebDriver driver = new ChromeDriver();   // 1. open a browser
driver.get("https://the-internet.herokuapp.com/login");  // 2. go to a page

String title = driver.getTitle();        // 3. read something back
System.out.println(title);               // "The Internet"

driver.quit();                           // 4. close everything
```

`new ChromeDriver()` launches Chrome and hands you a `driver` object — your
remote control for that browser. `driver.get(url)` navigates and *blocks* until
the page loads, so the next line always runs against a loaded page.

## Try it — read what is on the page

Once the page is open you can ask it questions. `driver.getTitle()` gives the
tab title; `driver.getCurrentUrl()` gives the address bar. To read visible text
you first grab an element and call `getText()`:

```java
WebElement banner = driver.findElement(By.tagName("h2"));
System.out.println(banner.getText());    // "Login Page"
```

You will meet `findElement` properly in the next lesson — for now, notice the
pattern: navigate, then read.

## Do it — a written exercise

Write out, line by line, a Selenium test that opens
`https://the-internet.herokuapp.com`, prints the page title, then closes the
browser. Label each line with which of the four moves it performs (open, go,
read, close). Then explain in one sentence what would go wrong if you forgot the
final `driver.quit()` and ran the test a hundred times in a row on a CI machine.
