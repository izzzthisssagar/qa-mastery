## A robot that runs your tests, not a robot that thinks

As a manual tester you read a test case, click through the app, and decide
whether what you see is right. **Test automation** hands the clicking to code:
a script repeats your steps the same way every run, checks the result against
what you told it to expect, and reports pass or fail. That is the whole trick —
and also the whole limitation. Automation checks **what you asked it to check**,
never what you forgot to ask.

## See it — what the machine is good (and bad) at

| Great for automation | Keep it manual |
| --- | --- |
| Regression suites run every build | Exploratory testing |
| Smoke checks after each deploy | Usability and "does this feel right?" |
| Same case across 50 data rows | One-off checks you'll never repeat |
| Tedious, deterministic, high-repeat | Visual polish and look-and-feel |

A machine never gets bored repeating the same login 200 times — that is its
superpower. But it cannot notice that the button is *ugly*, that the flow is
*confusing*, or that a feature nobody asked for is missing. Those need a human.

## Try it — spot the flaky test

A **flaky test** passes and fails on identical code, with no change between runs.
Usual causes: timing (the test clicks before the page loads), test order
(test B only passes if test A ran first), or shared state (a leftover record from
a previous run). Flakiness is worse than a coverage gap — once a suite "fails
sometimes," people stop trusting *any* red result, and real bugs get waved through.

Ask of any candidate test: *would this fail on good code?* If yes, fix the test,
not the app.

## Do it — a written exercise

Pick three checks from your manual QA work. For each, write one sentence on
whether you would automate it and why, judging **ROI**: cost to write plus
maintain, versus how often it runs and how costly the bug would be. A login
regression run every build earns its keep; verifying a marketing banner "looks
nice" does not. Now name one thing on your list that should stay manual forever —
and say what a human notices there that a script never will.
