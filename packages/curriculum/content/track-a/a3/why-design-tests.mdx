## You cannot click your way to confidence

Open any signup form. The email field alone accepts billions of strings; add a password, a name, a checkbox, and the number of possible inputs is effectively infinite. You will never test them all. So the real skill isn't clicking more, it's choosing which handful of inputs actually tell you something. Test design is how you carve an infinite space into a small, deliberate set.

## See it — same form, two approaches

Picture the BuggyShop signup form: name, email, password, age, terms checkbox. Compare 15 minutes of each approach.

| | Random clicking | Designed set |
|---|---|---|
| What you do | Type "asdf", mash buttons, retry | Pick representatives per field, by rule |
| Coverage | Whatever you happened to hit | Known: every input class touched |
| Repeatable? | No, different every run | Yes, written down step by step |
| Found a bug? | Hard to reproduce | Exact inputs recorded |
| Typical count | 30 messy attempts | 8 to 10 sharp cases |

Fewer steps, more coverage, and a paper trail. That is the whole pitch.

## Try it — reason through the password field

BuggyShop says passwords must be 8 to 20 characters. Random clicking might try "password", "12345678", "hunter2" — three valid-length strings that all test the same thing. A designer asks: what classes exist? Too short (7), valid (12), too long (21), and empty. Four inputs, four distinct behaviors probed. Notice you also test 7 and 21 on purpose — the edges, where off-by-one bugs hide. Random typing almost never lands exactly on 7 or 21.

## Do it — the contrast drill

On the BuggyShop signup form, run two timed passes.

1. Pass A: free-click for five minutes. No plan, no notes. When done, write down what you think you covered.
2. Pass B: list the fields. For each, jot the input classes and the edges (empty, too short, valid, too long, wrong format). Build 8 to 10 cases from that list and execute them, noting pass/fail.

Compare. Pass B will almost always find a defect Pass A missed, in fewer steps, and you can hand the case list to someone else to rerun. That reproducibility is the point. A guided interactive version of this drill is planned for a later release.
