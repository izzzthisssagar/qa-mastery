## Now make the page do things

You can find elements; now you act on them. The verbs are few and they map
directly onto what a human does: click, type, erase, read, and check whether the
control is even available. Master these five and you can drive most forms.

## See it — click, type, clear

```java
WebElement user = driver.findElement(By.id("username"));
user.clear();                 // empty the field first
user.sendKeys("tomsmith");    // type like a user

driver.findElement(By.id("password")).sendKeys("SuperSecretPassword!");
driver.findElement(By.cssSelector("button[type='submit']")).click();
```

`sendKeys` **appends** — it does not replace. If a field is pre-filled, typing
into it gives you the old text plus the new. That is why `clear()` comes first
whenever you need a known starting state.

## Try it — read and check state

```java
WebElement flash = driver.findElement(By.id("flash"));
System.out.println(flash.getText());   // "You logged into a secure area!"

WebElement btn = driver.findElement(By.cssSelector("button"));
boolean visible = btn.isDisplayed();   // is it on screen?
boolean usable  = btn.isEnabled();     // is it clickable, not greyed out?
```

`getText()` returns the **visible** text, trimmed — what the user reads, not the
raw HTML. `isDisplayed()` and `isEnabled()` answer different questions: a Submit
button can be *displayed* yet *disabled* until you tick a checkbox. Asserting on
both is how you catch a form that looks right but will not let the user proceed.

## Do it — a written exercise

Write the steps to log in at
`https://the-internet.herokuapp.com/login`: locate and clear the username,
type `tomsmith`, type the password, click submit, then read the flash message
with `getText()`. For one of the fields, explain what value you would end up
typing if you skipped `clear()` on a pre-filled field. Finish by describing one
real scenario where `isDisplayed()` returns true but `isEnabled()` returns
false.
