## Some things live outside the normal page

Native dropdowns, JavaScript alerts, iframes and pop-up windows each need a
special move. The common thread: by default the driver only sees the main
document, so for alerts, frames, and windows you must **switch context** before
you can touch them.

## See it — the Select class for dropdowns

A real HTML `<select>` is not clicked option-by-option; wrap it in `Select`:

```java
Select dropdown = new Select(driver.findElement(By.id("dropdown")));
dropdown.selectByVisibleText("Option 2");
dropdown.selectByValue("1");
dropdown.selectByIndex(2);
```

`Select` only works on a genuine `<select>` tag. Custom dropdowns built from
`<div>`s are just clickable elements — handle those with ordinary `click()`.

## Try it — alerts, frames, windows

A JavaScript alert blocks the page until you respond. Switch to it:

```java
Alert alert = driver.switchTo().alert();
System.out.println(alert.getText());
alert.accept();    // OK   (dismiss() = Cancel)
```

An iframe is a page inside the page — invisible to the driver until you enter it:

```java
driver.switchTo().frame("mce_0_ifr");   // by name/id, index, or WebElement
driver.findElement(By.id("tinymce")).sendKeys("hello");
driver.switchTo().defaultContent();      // step back out
```

A click that opens a new window leaves you on the old one. Find the new handle
and switch:

```java
String original = driver.getWindowHandle();
for (String handle : driver.getWindowHandles()) {
    if (!handle.equals(original)) driver.switchTo().window(handle);
}
```

Forgetting to switch back to `defaultContent()` or the original window is a
classic cause of "element not found" that has nothing to do with the locator.
