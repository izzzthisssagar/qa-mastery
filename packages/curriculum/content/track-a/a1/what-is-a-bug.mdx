## Error, defect, failure — not the same word

These get used interchangeably in conversation, but a tester keeps them straight:

- **Error (mistake)** — a *human* slip. A developer writes `>` meaning `>=`.
- **Defect (bug)** — the resulting flaw *in the code or document*.
- **Failure** — the *observable* wrong behaviour when that defect executes.

The chain: **error → defect → failure.** A developer's mistake plants a defect;
the defect becomes a failure only when the code runs under the conditions that
trip it. *(A future animation will play this chain end to end.)*

## See it — a defect that doesn't fail (yet)

A discount only miscalculates for orders over $10,000. The defect is real and
present in every build — but no test order is that large, so there's no failure.
Ship it, a whale customer buys, and the dormant defect finally fails in
production. **No failure observed ≠ no defect present.**

## Try it — the bug lifecycle

When you report a bug, it travels a little state machine:

`New → Assigned → In-progress → Fixed → Retest → Closed`

…with two important detours:

- **Reopened** — you retest the "fix" and it's still broken. Back it goes.
- **Rejected / Won't fix / Duplicate** — triage decides it isn't actionable.

Knowing the states tells you what your job is at each one (e.g. at *Retest*, you
verify the fix and check for regressions nearby).
