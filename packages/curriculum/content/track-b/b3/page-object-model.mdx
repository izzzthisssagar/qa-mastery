## Stop scattering locators across your tests

Write enough tests the naive way and the same `By.id("user-name")` appears in
ten methods. The day the developers rename that field, ten tests break and you
fix the same locator ten times. The Page Object Model fixes this by giving each
page its own class — one home for that page's locators and actions.

## See it — a LoginPage class

```java
public class LoginPage {
    private final WebDriver driver;
    private final By username = By.id("user-name");
    private final By password = By.id("password");
    private final By loginBtn = By.id("login-button");

    public LoginPage(WebDriver driver) { this.driver = driver; }

    public void login(String user, String pass) {
        driver.findElement(username).sendKeys(user);
        driver.findElement(password).sendKeys(pass);
        driver.findElement(loginBtn).click();
    }
}
```

The locators and the steps live together, encapsulated. Nothing outside this
class knows the field id is `user-name`.

## Try it — the test gets shorter and clearer

The test now reads like a sentence and holds no locators at all:

```java
@Test
public void validLogin() {
    LoginPage loginPage = new LoginPage(driver);
    loginPage.login("standard_user", "secret_sauce");
    assertTrue(inventoryPage.isLoaded());
}
```

Two payoffs. **Maintenance:** when `user-name` becomes `username`, you change
one line in `LoginPage` and every test keeps working. **Readability:** the test
describes *what* it does, not *how*. Note where the assertion lives — in the
**test**, not the page object. The page object drives the UI; the test decides
pass or fail.

## Do it — a written exercise

Design a `SearchPage` page object in plain text: list the locators it would
hold (search box, search button, results count) and the action methods it would
expose (e.g. `search(term)`, `getResultCount()`). Then write one or two
sentences on why an assertion like `assertEquals(getResultCount(), 5)` belongs
in the test and not inside the page object, and name one maintenance win POM
gives you when the search box's id changes.
