## When click and sendKeys are not enough

Hovering to reveal a menu, dragging one box onto another, right-clicking for a
context menu, holding Ctrl while pressing a key — none of these are a single
`click()`. The **Actions** class builds a *sequence* of mouse and keyboard
moves, then fires it all at once.

## See it — hover and the perform() rule

```java
Actions actions = new Actions(driver);
WebElement menu = driver.findElement(By.id("nav"));
actions.moveToElement(menu).perform();   // hover to reveal the submenu
```

Every Actions method only **queues** a step. Nothing happens until `perform()`
runs the chain. Leaving off `perform()` is the number-one Actions mistake: no
error, no effect, just a test that quietly did nothing.

## Try it — drag-drop, right-click, key combos

```java
// drag and drop
WebElement src = driver.findElement(By.id("column-a"));
WebElement dst = driver.findElement(By.id("column-b"));
new Actions(driver).dragAndDrop(src, dst).perform();

// right-click (context menu)
new Actions(driver).contextClick(src).perform();

// double-click
new Actions(driver).doubleClick(src).perform();

// Ctrl+A: hold modifier, send key, release modifier
new Actions(driver)
    .keyDown(Keys.CONTROL).sendKeys("a").keyUp(Keys.CONTROL)
    .perform();
```

`Keys` holds the non-printing keys — `CONTROL`, `SHIFT`, `ENTER`, `TAB`,
`ARROW_DOWN`. For a combo you `keyDown` the modifier, send the letter, then
`keyUp` to release it, all in one chain ended by `perform()`.

Write out, in words, an Actions chain that hovers over a navigation item to
reveal its dropdown, then clicks a submenu link inside it. Separately, describe
the four steps to send `Ctrl+C` to copy a selected field. For each example, name
the exact point where `perform()` must be called, and explain in one sentence
what symptom you would see if you forgot it.
