## You cannot act on what you cannot find

Every click, every typed character, every text read starts by locating an
element. Selenium gives you a family of **locator strategies** through the `By`
class, and one decision per element: do you want the first match, or all of
them?

## See it — the By locators

```java
driver.findElement(By.id("username"));            // by id attribute
driver.findElement(By.name("password"));          // by name attribute
driver.findElement(By.cssSelector("button.radius"));   // CSS selector
driver.findElement(By.xpath("//button[@type='submit']")); // XPath
```

`By.id` is the gold standard — ids are meant to be unique, so the match is exact
and fast. `By.cssSelector` is the everyday workhorse: `#username`, `.btn`,
`input[name='email']` all read cleanly. `By.xpath` is the most powerful (it can
match on text and walk upward), but long absolute paths like
`/html/body/div[2]/form/input[1]` shatter the instant the page layout changes.

## Try it — one element or many

```java
WebElement first = driver.findElement(By.tagName("a"));   // first link only
List<WebElement> all = driver.findElements(By.tagName("a")); // every link
System.out.println(all.size());
```

`findElement` returns the **first** match and throws
**`NoSuchElementException`** when nothing matches. `findElements` returns a
**List** — every match, or an *empty* list when there are none. So when an
element might legitimately be absent (a "no results" page), use `findElements`
and check `.isEmpty()` instead of wrapping `findElement` in a try/catch.
