## Code that makes decisions

A test that does the same thing no matter what it sees isn't a test — it's a
script. Real automation **branches**: if the login succeeded do this, otherwise
fail loudly. That branching is **control flow**, and it all runs on the booleans
from the last lesson.

## See it — if / else

```java
if (driver.findElements(By.id("error")).size() > 0) {
    System.out.println("Login failed as expected");
} else {
    System.out.println("No error shown");
}
```

The condition in the parentheses must be a `boolean`. If it's `true`, the first
block runs; otherwise the `else` block. You can chain `else if` for more cases.
This is how a test reacts to what the page actually did instead of blindly
marching on.

## Try it — loops over elements

When you have many things to check, you loop. A **for** loop is for known counts;
a **while** loop repeats until a condition flips.

```java
// for: check every row in a results table
for (int i = 0; i < rows.size(); i++) {
    System.out.println("Row " + i + ": " + rows.get(i).getText());
}

// while: keep polling until the spinner disappears
while (spinner.isDisplayed()) {
    // wait and re-check
}
```

The `for` loop has three parts: start (`int i = 0`), condition (`i < rows.size()`),
and step (`i++`). The `while` loop just repeats while its boolean holds.

## break and continue

Inside a loop, `break` exits immediately — useful once you've found the row you
wanted. `continue` skips the rest of *this* iteration and jumps to the next —
useful to ignore rows that don't matter.

```java
for (int i = 0; i < rows.size(); i++) {
    if (rows.get(i).getText().isEmpty()) continue;  // skip blanks
    if (rows.get(i).getText().equals("Widget")) break; // found it, stop
}
```

Combine conditions with `&&` (both true) and `||` (either true), and negate with
`!`.
