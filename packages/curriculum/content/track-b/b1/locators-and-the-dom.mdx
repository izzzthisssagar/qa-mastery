## The page is a tree, not a wall of text

A web page arrives as HTML, but the browser turns it into a living tree of
objects called the **DOM** (Document Object Model). Every heading, input, and
button is a **node** with a parent and children. Selenium never reads the raw
HTML — it walks this tree. So before a test can click a button, it has to *find*
that button's node. The query that finds it is called a **locator**.

## See it — the tree and its locators

```
  form
   ├─ input  id="email"   name="email"
   ├─ input  id="pass"    name="password"
   └─ button id="submit"  class="btn btn-primary"
```

Four common ways to point at the submit button:

| Strategy | Example | Note |
| --- | --- | --- |
| **id** | `#submit` | Best when present — meant to be unique |
| **name** | `name="..."` | Common on form fields |
| **CSS** | `button.btn-primary` | Fast, readable |
| **XPath** | `//button[@id='submit']` | Powerful, can match by text |

## Try it — uniqueness and stability

Two qualities make a locator good. First, **uniqueness**: it must match exactly
*one* element. `button.btn` might match ten buttons; `#submit` matches one.
Second, **stability**: it should survive everyday changes to the page.

Compare these for the same button:

- `#submit` — stable; the id rarely changes.
- `/html/body/div[2]/form/button[1]` — **brittle**. Add one `<div>` anywhere
  above and this absolute path points at the wrong node, or nothing.

A brittle locator breaks even though the element is still right there — that is
the difference between a test that fails because of a *bug* and one that fails
because of a *layout tweak*.

## Do it — a written exercise

Open any login page and inspect the password field in your browser's dev tools.
Write down two different locators that would find it: one stable (prefer an `id`
or `name`) and one deliberately brittle (a long position-based XPath). Then, for
the brittle one, name a small page change that would silently break it while the
field itself stays put. Finish with a one-line rule you will follow when
choosing locators in your own Selenium tests.
